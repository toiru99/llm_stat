# 무엇을 측정하나

**최신 물리학 연구 수준의 추론** 능력을 측정한다. 이름은 **C**omplex **R**esearch using **I**ntegrated **T**hinking – **P**hysics **T**est(발음 "critical point", 임계점). 표면적 패턴 인식으로 그럴듯한 답을 내는 단계에서, 실제 미공개 연구 문제를 진짜로 추론해 풀어내는 단계로의 **전환점(임계점)**을 측정하겠다는 의도가 이름에 담겨 있다.

핵심 질문은 셋이다. (1) **LLM이 학습 데이터를 넘어선 미공개 물리 연구 문제를 풀 수 있는가**, (2) **현실 물리 연구 워크플로에서 LLM이 어떤 모듈형 추론 과제를 도울 수 있는가**, (3) **물리 연구 맥락에서 LLM의 추론 과정·답을 신뢰할 수 있는가**. 자연어 과제와 달리 물리·수학 문제는 단 하나의 잘못된 추론이 전체 풀이를 무효화하는 "용서 없는(unforgiving)" 특성을 갖는다는 점에 주목한다.

# 과제 형식 / 예시

- **71개 복합 연구 챌린지(challenges)**: 입문 단계의 전체 연구 프로젝트를 모사한 자기완결형 문제. 각 챌린지는 *setup*(배경·맥락·가정 정의)과 *challenge question*(다단계 추론을 요하는 핵심 연구 질의)로 구성. 멘토가 신입 연구자를 온보딩하듯 범위와 가정을 명시한다.
- **190개 체크포인트(checkpoints)**: 복합 챌린지를 더 단순한 하위 과제로 분해한 것. 세분화된 진단을 가능하게 함. 협업 환경에서 팀원에게 분배될 법한 모듈 단위 과제를 모사.
- **개방형 Q&A** 형식. 답은 대부분 **수치 또는 기호식(symbolic expression)**. 너무 복잡해 SymPy로 다루기 어렵거나 등가 형태가 많으면 **테스트 케이스가 딸린 Python 함수**를 답으로 요구. 모든 기호·관례·물리 단위는 문제에 명시.
- 공개된 예시 챌린지는 "Quantum Error Detection"(양자 오류 검출) 1건이며, 나머지 70챌린지(테스트셋)는 오염 방지를 위해 **비공개**.

# 채점 방식

- **물리 특화 자동 채점 파이프라인**. LLM 답을 먼저 **구조화된 코드 블록으로 정규화**한 뒤, 수치값·SymPy 호환 기호식·테스트 케이스 실행으로 채점하는 **커스텀 스크립트**로 평가.
- 모든 답이 **기계 검증 가능(machine-verifiable)**하도록 설계. **물리적으로 의미 있는 오차 허용(error tolerance)**과 등가 형태를 반영.
- LLM 심판(LLM-as-judge)에 의존하지 않음 — 심판 자체의 능력 한계와 표면적 요인(프롬프트 문구·답 형식)에 대한 민감성을 피하기 위함.
- 온라인 평가 서버를 운영해 연구 커뮤니티의 독립 검증과 프런티어 모델 진척의 지속 모니터링을 지원.

# 점수 해석 (높음 / 낮음)

- **높음** = 정답이 정해진 시험을 넘어, 미답·미공개 연구형 문제를 다단계로 추론해 풀어내는 깊은 과학 추론에 강함. 신뢰할 만한 추론 흔적(reasoning trace).
- **낮음** = 잘 정형화된 문제엔 강해도, 개방형·희소해(sparse solution space)·깊은 의사결정 트리를 가진 연구급 문제 앞에서 무너짐.
- 추측 저항적 설계라 운으로 올라가는 기저선이 사실상 없음 — 낮은 점수는 곧 능력 한계.

# 난이도 · 포화 · 현 SOTA

- 물리학자들이 평가하기로, **현역 PI가 주니어 대학원생에게 줄 법한 "워밍업 연구 과제" 수준**의 난이도. 탄탄한 물리 훈련과 약간의 도메인 전문성이 필요하나 사려 깊은 탐색으로 접근 가능한 선.
- **극히 낮은 현 SOTA**: 베이스 모델 중 최고인 **GPT-5 (high)도 챌린지 평균 정확도 5.7%**, 대부분의 다른 모델은 0%에 가까움. **도구(코드 인터프리터·웹 검색) 사용 시 GPT-5 (high)가 12.6%**로 완만히 상승.
- 평가 대상에는 추론 특화 모델(GPT-5 high, o3, o4-mini, Gemini 2.5 Pro/Flash, DeepSeek R1, Claude Opus 4)과 범용 챗 모델(GPT-5 minimal, GPT-4o, Llama-4 Maverick)이 포함.
- 더 엄격한 평가 지표에서는 연구급 문제에 대한 신뢰성 결여가 더 드러남 → **포화와 거리가 멀고**, 프런티어 모델 변별에 폭넓은 여지가 남음.

# 한계 · 주의

- **단일 도메인(물리)**: 전 분야 추론의 대리지표는 아님. (레이더에서 GPQA·HLE와 함께 추론 축을 구성.)
- **극단적 난이도로 인한 저변별**: 현재 대부분 모델이 한 자릿수~0%라 *하위~중위* 모델 간 변별은 약함. 반대로 **상위 포화는 전혀 없어** 프런티어 변별에는 강함(GPQA와 정반대 위치).
- **물리 전문성 요구**: 채점·해석에 물리 배경이 필요. 다만 자동 채점·온라인 서버로 일반 개발자도 검증 가능하게 완화.
- **공개 예시 1건**: 투명성을 위해 1챌린지만 공개, 나머지는 비공개 → 외부 정성 분석은 제한적. (대신 오염 위험은 크게 낮춤.)
- 이론 비중이 높음(챌린지 80.3%) → 실험·계산 flavor 표본은 상대적으로 적음.

# 다른 벤치마크와의 관계

- [[gpqa-diamond]] GPQA Diamond — 박사급 "객관식" 과학 지식·추론. CritPt는 그보다 **한 단계 위**로, **객관식이 아니라 개방형 미공개 연구 문제**를 다뤄 추측·암기 회피가 본질적이고 훨씬 깊다(GPQA는 상위 포화, CritPt는 바닥).
- [[humanitys-last-exam]] HLE — 초고난도 다분야. CritPt는 물리 단일이지만 "연구 워크플로 모사"라는 점에서 결이 다름.
- 답 형식 설계는 SciCode·FrontierMath·TPBench에서 부분적으로 영감.

# 레이더 기여 근거

- **reasoning (1.0)**: 본질이 미공개 물리 연구 문제에 대한 다단계 원본 추론(original reasoning) → 추론 축에 단일·최대 기여. 표면 패턴 재조합이 아니라 진짜 추론을 가리는 설계라 "추론" 축의 상한선 신호로 적합. 물리 단일 도메인이므로 knowledge 등 부 기여는 두지 않음(좁은 범위 반영).

# 한국 · 앨런 관점

연구·R&D 최전선에서 모델이 "진짜로 새로운 문제를 추론해 푸는가"를 가늠하는 가장 빡센 축. 정형 시험이 아니라 미답 문제라, 과학·공학 연구 보조 용도에서 모델의 실질 한계를 보수적으로 보여 준다. 점수가 한 자릿수라는 사실 자체가 "현재 LLM은 연구급 물리 추론을 아직 신뢰할 수 없다"는 메시지.
