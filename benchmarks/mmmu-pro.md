# 무엇을 측정하나

멀티모달 모델이 **이미지와 텍스트를 함께 보고 통합적으로 이해·추론**하는 능력을 측정한다. MMMU-Pro는 기존 MMMU의 "더 견고한(robust)" 버전으로, 핵심 문제의식은 하나다 — *현재 벤치마크 점수가 진짜 시각 이해를 반영하는가, 아니면 모델이 통계적 지름길로 정답을 맞히는가?* 논문은 두 가지 우회로를 지목한다. (1) **텍스트 단독 의존(Text-Only Dependency)** — 이미지 없이도 풀리는 문항, (2) **지름길 악용(Shortcut Exploitation)** — 이미지가 필요한 문항조차 보기 안의 단서·사전지식 상관으로 맞히는 경우. MMMU-Pro는 이 두 우회로를 막아 모델이 실제로 **"보고(see)" 동시에 "읽어야(read)"** 풀리도록 강제한다.

# 과제 형식 / 예시

- 30개 과목, 6개 핵심 학문 분야에 걸친 **대학 수준 객관식**. 차트·도표·지도·화학구조식 등 다양한 이미지 유형 포함.
- **3단계 제작 파이프라인**(논문 Figure 1):
  1. **텍스트만으로 풀리는 문항 제거** — 강한 텍스트 전용 LLM 4종(Llama3-70B, Qwen2-72B, Yi-1.5-34B, Mixtral-8x22B)에게 이미지 없이 10회 풀게 해, 다수가 반복해서 맞히는 문항을 걸러냄.
  2. **보기 확장(4 → 최대 10지선다)** — GPT-4o 생성 + Claude 3.5 필터링 + 사람 2회 검수로 후보 보기를 늘려 추측 여지를 차단.
  3. **비전 전용(Vision-only) 설정 추가** — 질문 텍스트를 스크린샷·사진 **이미지 안에 박아** 넣어, 텍스트 입력을 따로 주지 않음(폰트·배경·글꼴 크기를 사람이 직접 다양화해 촬영). 사용자가 화면을 캡처해 그대로 던지는 실제 사용 양상을 모사.
- 최종 규모: **총 3,460문항 = 표준 형식 1,730 + 스크린샷/사진(Vision) 1,730**.
- 예시(논문 Figure 4): "그림 속 박테리오파지 감염의 5단계에 라벨을 붙여라" 같은 문항이 최대 10개 보기와 함께 스크린샷 안에 들어가 출제됨.

# 채점 방식

- 정답률(accuracy), 자동 채점. 10지선다라 **무작위 추측 기저선은 ~12%**(논문 Random Choice 12.4~12.8%).
- **MMMU-Pro 종합 점수 = (2) 표준 10지선다 설정 + (3) 비전 전용 설정 두 점수의 평균.** (1)단계의 4지선다나 원본 MMMU 검증셋 점수는 비교·참고용으로만 보고.
- Direct 프롬프트와 CoT 프롬프트 중 **더 높은 쪽**을 종합 결과로 보고.

# 점수 해석 (높음 / 낮음)

- **높음** = 차트·문서·다이어그램·사진 등 시각 자료를 보고 통합 추론하는 멀티모달 능력이 강함. 화면 캡처를 던졌을 때 텍스트와 시각요소를 함께 읽고 푸는 실무형 능력.
- **낮음** = 텍스트 단독엔 강해도 진짜 시각 이해·통합에서 무너짐. 특히 Vision 설정에서 점수가 더 떨어지면, 이미지 속 텍스트를 읽어도(OCR) 그 맥락·시각요소와의 관계를 추론하지 못한다는 신호.
- 12% 부근 = 사실상 추측.

# 난이도 · 포화 · 현 SOTA

- **전 모델이 원본 MMMU 대비 크게 하락**: 모델별 16.8~26.9% 하락(논문 Δ₃). 보기 확장(4→10)만으로도 GPT-4o가 64.7→54.0%로 10.7%p 빠지고, Vision 전용 설정에서 추가로 4.3%p 더 떨어짐.
- 발표 시점 기준 상위권도 낮음: GPT-4o(0513) 표준 10지선다 54.0% / Vision 49.7%, Claude 3.5 Sonnet 표준 55.0% / Vision 48.0%. 다수 오픈소스 모델은 Vision 설정에서 **10~30%대**에 머묾.
- 사람 전문가(High) 추정치는 표준 10지선다 85.4% / Vision 85.4% 수준으로, 모델과 큰 격차. → **현재도 미포화**이며 변별력이 넓게 살아 있는 벤치마크.

# 한계 · 주의

- **리더보드 커버리지 절반(약 48%)**: AA 리더보드에서 MMMU-Pro 열이 채워진 모델은 추적 대상의 **절반가량(~48%)뿐**이다. 멀티모달은 **단일 소스 축**이라(다른 축에서 끌어오는 교차 보간 없음), 미측정 모델은 멀티모달 축이 그대로 **결측(null, 다각형 점선·회색)** 처리된다. 즉 카드에 멀티모달 점이 없다고 능력이 0이라는 뜻이 아니라 *측정 자체가 안 된 것*이며, 모델 간 멀티모달 비교는 절반 표본 위에서만 성립함에 유의.
- **객관식 한계**: 추측·소거 여지가 남음(10지선다로 기저선을 12%까지 낮춰 완화).
- **OCR 프롬프트는 효과 미미**: 강한 모델은 이미 텍스트 추출을 내재화. 단, 높은 OCR 정확도가 곧 강한 멀티모달 추론은 아님(LLaVA-OneVision-72B는 OCR은 높아도 Vision 추론은 뒤처짐). 진짜 병목은 **시각·텍스트 통합 추론**.
- **CoT 의존성·편차**: CoT가 대체로 도움되나 분야별 편차가 크고, 지시 따르기가 약한 모델은 CoT로 오히려 손해(VILA-1.5-40B 등).
- **오류 구성**: GPT-4o의 Vision 오류 60건 분석에서 **추론 오류 46%**, 지각(perceptual) 오류 27%, 지식 오류 25%, OCR 오류 0%. 텍스트 인식이 아니라 통합·추론이 주 실패 지점.

# 다른 벤치마크와의 관계

- 멀티모달 축의 핵심 소스. 텍스트 전용 추론·지식 벤치마크와 상보적.
- [[gpqa-diamond]] GPQA Diamond — 텍스트 기반 과학 추론(추론·지식 축). MMMU-Pro는 같은 "대학원/대학 수준 난이도"를 **시각 의존성**으로 옮긴 격.
- 원본 MMMU 대비: 동일 도메인 구성이되 텍스트 우회·추측을 막고 Vision 설정을 추가해 난이도·견고성을 끌어올린 상위 호환.

# 레이더 기여 근거

- **multimodal (1.0)**: 본질이 이미지+텍스트 통합 이해·추론이며, 텍스트 단독 풀이를 설계적으로 차단했으므로 멀티모달 축에 **단독 주 기여**. 다른 축으로 분산하지 않음(시각 이해 신호를 희석하지 않기 위해). 단, 단일 소스이고 커버리지가 절반이라 결측 모델이 많다는 점은 위 한계에 명시.

# 한국 · 앨런 관점

화면 캡처(스크린샷)·문서 사진·차트를 그대로 던져 묻는 실사용 패턴과 가장 가까운 멀티모달 지표. 보고서 도표 해석, 화면 캡처 기반 질의, 문서 이미지 처리 같은 업무형 사용에서 모델이 믿을 만한지 가늠하는 축이다. 다만 절반의 모델만 측정돼 있어, 카드에 점이 없으면 "약하다"가 아니라 "미측정"으로 읽어야 한다.
