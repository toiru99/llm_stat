# 무엇을 측정하나

GDPval은 AI가 **실제 경제적 가치가 있는 직무 작업**을 끝까지 해내는 능력을 측정한다. 학술 시험식 추론 난이도(GPQA·MMLU류)가 아니라, 미국 노동통계국(BLS)·O\*NET 직무 데이터에 기반한 **현업 전문가의 실제 업무 산출물(deliverable)**을 모델이 만들게 하고, 그 결과물을 인간 전문가의 결과물과 직접 비교한다.

설계 출발점이 경제 자체다. 저자들은 미국 GDP 기여 상위 산업을 먼저 고르고, 그 안에서 임금·보상이 가장 큰 **"주로 디지털(predominantly digital)"** 지식노동 직종을 추렸다. 즉 "모델이 얼마나 똑똑한가"가 아니라 **"모델에게 진짜 돈 되는 일을 시킬 수 있는가"**를 묻는, 노동시장 영향 측정용 벤치마크다.

# 과제 형식 / 예시

- 각 과제 = **요청(request, 참고 파일 포함)** + 기대 **산출물(deliverable)** 한 쌍.
- 산출물은 단순 텍스트가 아니라 실무 포맷 전반: 문서, 슬라이드 덱, 스프레드시트(.xlsx), PDF, 다이어그램, CAD 설계 파일, 사진·영상·오디오, 소셜 포스트, 고객지원 대화 등 **멀티모달**.
- 참고 파일이 많다: **골드 서브셋 최대 17개 / 전체 세트 최대 38개** 파일을 읽고 종합해야 한 과제를 푼다.
- 한 과제의 인간 작업량이 **평균 약 7시간**, 고난도는 수 주에 걸친다 → "장기 호흡(long-horizon)" 과제.

논문 Figure 1의 실제 예시:
- 제조 엔지니어: 조립 라인용 케이블 릴 스탠드의 3D 모델 설계.
- 금융·투자 분석가: 라스트마일 배송 경쟁 구도 분석 작성.
- 등록 간호사: 피부 병변 이미지 평가 + 상담 리포트 작성.
- 영상 편집자: 영상·오디오가 포함된 고에너지 인트로 릴 제작.
- 고객 서비스: 반품 요청 불만 고객 응대 이메일.
- 컨시어지: 4인 가족 바하마 일주일 럭셔리 여행 일정 설계.
- 부동산 에이전트: 신축 매물 판매 브로슈어 디자인.

# 채점 방식 (원본 vs AA Elo)

## 원본(OpenAI) — 인간 전문가 쌍대비교 승률
- **헤드라인 지표는 head-to-head 인간 전문가 비교**다. 해당 직종 전문가에게 요청·참고 파일과 **익명(라벨 제거)된 두 결과물**을 제시하고 어느 쪽이 더 나은지 순위를 매기게 한다.
- 골드 220개 과제는 블라인드 쌍대비교로 채점하며, **비교 1건당 평균 1시간 이상**이 든다. 전문가는 정확성뿐 아니라 구조·스타일·포맷·미학·관련성 같은 주관적 요소까지 함께 본다.
- 산출물이 인간 결과물을 **이기거나(win) 비기면(tie)** parity(50%)에 가까워진다. "상한 없음" 설계라 모델이 강해져도 포화하지 않고, 필요하면 인간 기준선을 더 강한 모델로 갈아끼울 수 있다.
- 과제 생성 자체도 엄격하다: 전체 1,320개가 모델 스크리닝 + 다단계 인간 검토 파이프라인을 거쳤고, **과제당 평균 5회(최소 3회) 전문가 검토**를 받았다.

## 실험적 자동 채점기
- 골드 서브셋용으로 OpenAI가 별도 자동 채점 모델을 학습해 공개(evals.openai.com)했다. 인간 전문가와 **65.7% 일치**하며, 이는 인간 채점자끼리의 일치율 **70.8%보다 약 5%p 낮은** 수준이다(즉 인간 합의도 완벽하지 않다).

## AA 변형 (GDPval-AA v2)
- Artificial Analysis가 **공개 220개 과제를 독립 재평가**한 버전. AA가 직접 에이전트 환경(셸 + 웹 브라우징)에서 모델을 돌리고, 익명화한 산출물을 **LLM 심판이 쌍대비교(pairwise)**해 **Elo**로 환산한다(인간 기준선 1,000점 앵커).
- 리더보드 표기는 "Agentic Real-World Work Tasks", 정규화 점수 = (Elo − 500) / 2000. **현재 AA Intelligence Index에 에이전트 카테고리 최대 비중으로 포함**된다.
- 즉 원본은 *인간 심판 승률*, AA는 *LLM 심판 Elo*로 채점 주체와 척도가 다르다.

# 점수 해석 (높음 / 낮음)

- **높음** = 사무·전문직 실무를 사람 대신 끝까지 해내는 실무 산출 능력이 강함. parity(50%)에 가까울수록 "그 직종 전문가 결과물과 맞먹는다."
- **낮음** = 추론은 되더라도 지시 준수·포맷·파일 처리·완결성에서 무너져 결과물을 쓸 수 없음.
- 다른 벤치마크 중 **"이 모델로 실제 일을 시킬 수 있나"에 가장 가까운 지표**다.

# 난이도 · 현 SOTA

- **아직 누구도 parity(50%) 미달** → 미포화. 논문 골드 쌍대비교(Figure 5) 기준 인간 대비 승률(이기거나 비긴 비율):
  - Claude Opus 4.1 **47.6%** (최고, 특히 미학·포맷에서 강세)
  - GPT-5 high **38.8%** (정확성·지시 준수에서 강세)
  - o3 high **34.1%**, o4-mini high 27.9%, Gemini 2.5 Pro 25.5%, Grok 4 24.3%, GPT-4o 12.4%
- OpenAI 프런티어 모델 성능은 출시 시점에 따라 **거의 선형으로 향상**(GPT-4o → o3 → GPT-5).
- **추론 노력↑ / 컨텍스트↑ / 스캐폴딩↑이면 점수가 오른다**: 추론 effort를 low→high로 올리면 일관 상승하고, 프롬프트 튜닝만으로 GPT-5가 38.8%→43.1%로 올랐다. 프롬프트로 PDF의 검은 사각형 아티팩트를 제거하고 PowerPoint 포맷 오류율을 86%→64%로 낮추는 등, **쉬운 개선 여지가 많다** = 아직 한계가 아님.

# 한계 · 주의 (커버리지)

- **리더보드 커버리지가 낮다(~23%)**: AA가 추적하는 모델 중 GDPval-AA 점수가 있는 모델은 일부뿐이라, 에이전트 축이 **결측인 모델이 많다**. 레이더에서 이 축은 비교 대상이 좁아질 수 있다(radar-spec의 결측=null 규칙 적용, 교차 보간 없음).
- **채점이 주관적·비쌈**: 인간 쌍대비교는 비교당 1시간+, 채점자 합의율도 70.8%에 그친다. 자동 채점기는 빠르지만 정확도가 인간보다 낮다.
- **승자 식별 누출 가능성**: 익명화해도 문체로 모델 출처가 추정될 수 있다(예: em dash, 1인칭, Grok이 자기 언급). 저자도 이를 caveat로 명시.
- **범위 한정**: 골드 공개분은 220개뿐이고, 텍스트 위주 직종과 멀티모달 직종의 과제 성격이 다르다(저자 caveat). 전체 1,320개는 비공개.
- **현재 진행형 벤치마크**: parity 미달이라 변별력은 크지만, 채점·환경 설정(에이전트 셸/브라우징/best-of-N)에 따라 점수가 흔들릴 수 있다.

# 다른 벤치마크와의 관계

- 추론 시험형([[gpqa-diamond]] GPQA, [[humanitys-last-exam]] HLE)과 정반대 축: GDPval은 *난이도*가 아니라 *실무 완결성*을 본다.
- 코딩 에이전트류와 달리 단일 도메인이 아니라 **44개 직종 전반의 일반 사무·전문직 산출물**을 포괄.
- 레이더의 **에이전트(agentic) 축**을 대표하는 핵심 소스(도구 사용 + 다단계 + 멀티모달 파일 처리 + 완결된 산출물).

# 레이더 기여 근거

- **agentic (1.0)**: 본질이 "도구·파일을 다뤄 실무 산출물을 끝까지 만든다"로, 에이전트 축의 정의(도구 사용·다단계 실무 수행)와 정확히 일치 → 주 기여, 단일 축 가중치 1.0.
- 추론·지식이 일부 필요하지만, GDPval의 변별력은 *추론 난이도*가 아니라 *지시 준수·포맷·완결성·멀티모달 파일 처리*에서 나오므로(실패 모드 대부분이 instruction-following·formatting), 다른 축으로 분산하지 않고 에이전트 축에 집중한다.

# 한국 · 앨런 관점

"이 모델을 우리 업무에 실제로 투입할 수 있나"를 가장 직접적으로 보여주는 지표. 문서·보고서·슬라이드·스프레드시트 작성처럼 사무·전문직 산출물을 대신 만들게 할 때의 신뢰도와 직결된다. 다만 한국어·국내 직무 맥락이 아니라 미국 BLS 직종·영어 산출물 기준이라는 점, 그리고 리더보드 커버리지가 낮아 점수 없는 모델이 많다는 점을 감안해 해석해야 한다.
